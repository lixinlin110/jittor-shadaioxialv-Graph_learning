# 中期汇报材料

## 研究题目

基于图学习的动态推荐任务研究：面向时序图未来链接预测的时间感知候选节点重排序策略。

## 任务定义

真实推荐系统、社交网络、论文引用网络和网页跳转网络中的交互关系会随时间持续演化。项目将历史交互表示为 `(source, destination, time)` 三元组，在给定源节点、预测时间和候选目标节点集合时，预测未来最可能发生交互的目标节点顺序。

核心评价指标为 MRR，辅助指标包括 Recall@K、NDCG@K、Hit Rate、训练时长、推理时间和显存占用。

## 技术路线

```text
原始时序交互三元组
→ 字段检查与清洗
→ ID 映射与时间排序
→ 按时间划分训练/验证/测试集
→ 候选目标节点集合
→ 官方 CRAFT baseline
→ 时间窗口动态图快照
→ 时间衰减边权重
→ 近期邻居采样
→ 候选节点概率打分与重排序
→ MRR/Recall@K/NDCG@K/Hit Rate 评测
→ 消融分析与结果文件生成
```

## 基线结果

当前记录显示，官方 CRAFT baseline 在未修改代码的情况下，通过增加 dataset1 的训练轮数与 early stopping 上限，A 榜分数从 `1.042` 提升到约 `1.0878`。这一结果说明 baseline 已经具备可运行基础，但仍需要进一步规范随机种子、日志、checkpoint 和消融对照。

详细记录见 [experiments.md](experiments.md)。

## 计划中的改进

1. 时间窗口动态图快照：按时间切分历史交互，比较不同窗口长度下的模型效果。
2. 时间衰减边权重：让越接近预测时间的历史交互对候选节点打分贡献越大。
3. 近期邻居采样：围绕 `num_neighbors` 调整近期邻居序列，降低过早历史交互和热门节点偏置。
4. 候选节点重排序：将官方 baseline 得分与频次、重复交互、新连接、时间衰减信号融合。
5. 消融实验：对官方 baseline、仅时间窗口、仅时间衰减、仅近期邻居、综合策略进行对比。
